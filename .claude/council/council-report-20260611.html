🎯 خلاصة الرئيس

المشروع سليم معمارياً ويُترجَم بنجاح. ألم المالك ليس بناءً معطّلاً — سجلّات التعطّل مجرد نفاد ذاكرة لأدوات التحليل المحلية. المشكلة ثلاث فئات منفصلة:

  1. الأمن (أولاً، اليوم): سرّ JWT واحد ضعيف OS.009988.os للأدوار الثلاثة (يُزوّر أدمن خارق)، تسريب OTP في الاستجابة عند فشل SMS، شحن محفظة مجاني + تأكيد ذاتي، غياب حُرّاس الأدوار، تسريب محادثة (IDOR).
  2. صحّة المال/المنطق: عمولة المنصة = صفر دائماً، قبول مزدوج للرحلة، تسوية بلا معاملة (رحلات مجانية)، أسعار المزايدة بلا حدود.
  3. الإنتاج + UX: webhook الدفع يرفض كل الطلبات الحقيقية، لا فهارس DB، ~25 entity بلا migration، تطبيق السائق نصفه إنجليزي، أزرار ميتة، خريطة حية فارغة.

🥇 الخطوة الأولى: إصلاح سلسلة ثقة المصادقة — 3 أسرار JWT قوية منفصلة مع fail-fast (بلا قيم احتياطية مزروعة). كل ما عداه بلا معنى ما دام سلسلة نصية واحدة تُزوّر أدمن خارق.

✅ حيث يتّفق المجلس (ثقة عالية — وجدها وكيلان مستقلان أو أكثر)

  • CRITتسريب OTP في الاستجابة عند فشل SMS — استيلاء كامل على الحساب. (المعارض + التوسّعي)
  • CRITconfirmWalletRecharge يتيح للراكب شحن محفظته ذاتياً — مال مجاني. (المعارض + المبادئ)
  • CRITالتسوية مبتلَعة → رحلات مجانية برصيد محفظة غير كافٍ. (المعارض + المبادئ)
  • HIGHلا فهارس على Order/Driver/Rider/Bid — مسح كامل عند كل تسجيل دخول + cron كل دقيقة. (المبادئ + التوسّعي)
  • HIGHالخريطة الحية فارغة — NEXT_PUBLIC_GOOGLE_MAPS_KEY غير معرّف في أي ملف. (الغريب + التوسّعي)
  • HIGHمفتاح خرائط مزروع في git (.env + كلا manifest أندرويد). (المعارض + الغريب)
  • CRITسرّ JWT ضعيف مشترك = كلمة مرور الأدمن أيضاً. (المعارض)

⚔️ تصحيحات المجلس للفرضيات

  • "التطبيقات لا تعمل": الغريب + المنفّذ يدحضان — سجلّات التعطّل نفاد ذاكرة أدوات محلية، لا أعطال. flutter analyze = 0 أخطاء في التطبيقين، وكل بوابات البناء خضراء. الإحساس بالعطل سببه أزرار ميتة + تطبيق سائق نصفه إنجليزي + خريطة فارغة.
  • خطأ live/page.tsx المعروف: المنفّذ يدحض — مُصلَح بالفعل (cast في live/page.tsx:36)، tsc 0 أخطاء.

🔍 نقاط عمياء كشفتها المراجعة المتبادلة

  • مسار الشحن المجاني "المحاكى" موجود لأن webhook الدفع الحقيقي معطّل (HMAC على JSON معاد تحليله يرفض كل webhook حقيقي) — إصلاح أمن الشحن يستلزم إصلاح rawBody أيضاً.
  • providerShare لا يُعيَّن أبداً → حتى بعد عمل الدفع، المنصة تدفع للسائق 100% وتربح صفراً.
  • ignoreBuildErrors:true + ignoreDuringBuilds:true في اللوحة → أخطاء النوع/اللينت تُشحَن صامتة. أخضر ≠ آمن.
  • ~25 entity بلا migration (incl. pricing_zone/fleet) → إنشاء الطلب يرمي "relation does not exist" في DB إنتاج نظيف.
